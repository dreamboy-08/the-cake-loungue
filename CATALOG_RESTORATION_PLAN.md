# Product Catalog Restoration Plan

## 1. Executive Summary
This document outlines the strategy for restoring the visual integrity of the product catalog. Currently, the catalog consists of **342 products** across **57 categories**, but a staggering **93.6% (320 products)** share a limited set of only **13 placeholder images**. This plan proposes a transition to a high-quality catalog utilizing at least **57 unique images** tailored to specific categories and flavors.

## 2. Current State Audit
*   **Total Products:** 342
*   **Total Categories:** 57 (Average 6 products per category)
*   **Unique Images Currently:** 13
*   **Primary Issue:** High duplication. For example, one placeholder image (`photo-1561758033-7e924f619b47`) is used by 122 different products across categories like "Premium Cakes", "Luxury Cakes", and "Festival Cakes".

## 4. Restoration Mapping Strategy

### 4.1 Keyword-Flavor Alignment
We will map products to images based on primary flavor keywords in their name and `flavor` attribute.
*   **"Chocolate" / "Truffle" / "Fudge":** Map to high-contrast chocolate ganache and textured cocoa images.
*   **"Berry" / "Strawberry" / "Raspberry":** Map to vibrant red/purple fruit-topped cakes.
*   **"Vanilla" / "Almond" / "White":** Map to clean, bright, ivory and pastel-toned cakes.
*   **"Mango" / "Citrus" / "Lemon":** Map to warm yellow and orange-toned fruit cakes.
*   **"Saffron" / "Rasmalai":** Map to Indian-fusion styled cakes with pistachio and petal garnishes.

### 4.2 Category-Specific Visuals
*   **Cakes in Jars/Boxes:** Use images that clearly show the container (glass jars, premium boxes).
*   **Interactive Cakes (Pinata/Pull Me Up):** Use "action" shots showing the shell or the drip effect.
*   **Celebration Categories (Wedding/Graduation):** Use professional, high-end photography featuring appropriate props (mortarboards, floral lace).

### 4.3 Technical Implementation
1.  **JSON Mapping:** Utilize `src/constants/proposed_images.json` as the source of truth for the 1-to-1 or 1-to-many product-to-image mapping.
2.  **Remote Optimization:** All images must use the `images.unsplash.com` remote pattern for Next.js image optimization.
3.  **Audit Persistence:** Log all changes in a dry-run first using the `bulkImageUpdate.ts` utility before applying to Firestore.

## 5. Unique Image Estimates
*   **Phase 1 (Basic Restoration):** **57 unique images**. This ensures each of the 57 categories has its own distinct visual identity, reducing duplication from 13 images to 57.
*   **Phase 2 (Premium Restoration):** **114 unique images**. This would provide 2 unique images per category, allowing for flavor-based differentiation within a category (e.g., separating "Blueberry Cheesecake" from "Oreo Cheesecake").
*   **Phase 3 (Total Unique Catalog):** **342 unique images**. Full 1-to-1 mapping for a bespoke boutique experience.

## 6. Conclusion
The initial restoration will focus on **Phase 1 (57 unique images)**, which represents a **438% increase in visual variety** and significantly improves the professional appearance of the catalog.
